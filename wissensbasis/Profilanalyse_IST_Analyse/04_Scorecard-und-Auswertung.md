# Säule 4 - Scorecard und Auswertung: Vom Befund zur Übergabe

Diese Säule führt die Einzelbefunde aus Säule 01 bis 03 in ein bewertbares Gesamtbild zusammen, priorisiert die größten Baustellen und übergibt sauber an die Positionierungs-Phase. Wichtig bleibt die Trennung: Die Scorecard bewertet den IST-Zustand, sie formuliert keine Strategie. Die Übergabe markiert genau die Schnittstelle.

---

## 1. Bewertungsraster

Bewerte jede Dimension auf einer einheitlichen Skala. Eine 4-stufige Skala zwingt zur Entscheidung (kein bequemes Mittelfeld):

| Stufe | Bedeutung |
|---|---|
| **0 - fehlt** | Element gar nicht vorhanden |
| **1 - schwach** | vorhanden, aber deutlich unter Standard |
| **2 - solide** | erfüllt den Standard, kein Auffälligkeit |
| **3 - stark** | über Standard, Vorbildcharakter |

Bewertet werden die Dimensionen aus den vorherigen Säulen, je Plattform:

| Dimension (Quelle) | Plattform 1 | Plattform 2 | Plattform 3 | Notiz |
|---|---|---|---|---|
| Profil-Vollständigkeit (01) | | | | |
| Bio-Klarheit (01) | | | | |
| Konsistenz über Kanäle (01) | | | | |
| Visual-Kohärenz (01) | | | | |
| Tonalitäts-Konsistenz (01) | | | | |
| Posting-Frequenz (02) | | | | |
| Engagement-Rate vs. Benchmark (02/03) | | | | |
| Follower-Wachstum (02) | | | | |
| Outcome-Signale (02) | | | | |
| Position im Wettbewerb (03) | | | | |

**Regeln für faire Bewertung:**
- Jede Note braucht eine Notiz mit der zugrundeliegenden Beobachtung oder Zahl.
- Performance-Noten immer gegen einen Bezugswert (Benchmark oder eigene Vergangenheit), nie absolut.
- Keine Note ohne Datengrundlage. Wo Daten fehlen, „nicht bewertbar" eintragen, nicht raten.

---

## 2. Priorisierung der größten Baustellen

Aus dem Raster fallen viele Befunde. Nicht alle sind gleich wichtig. Priorisiere nach zwei Achsen: **Wirkung** (wie stark beeinflusst der Befund das Ergebnis) und **Aufwand** (wie teuer ist die Behebung). Das ergibt eine einfache Matrix:

| | Geringer Aufwand | Hoher Aufwand |
|---|---|---|
| **Hohe Wirkung** | **Quick Wins** (zuerst) | **Großprojekte** (planen) |
| **Geringe Wirkung** | **Nebenbei** (mitnehmen) | **Ignorieren** (vorerst) |

**Typische Quick Wins (hohe Wirkung, geringer Aufwand):**
- Falsche oder fehlende Links in Bios korrigieren
- Inkonsistente Handles vereinheitlichen, wo möglich
- Fehlende CTA in der Bio ergänzen
- Verwaiste Profile schließen oder reaktivieren
- Veraltete Highlights / Pinned Posts aktualisieren

**Priorisierungs-Tabelle:**

| Baustelle | Wirkung (1 bis 3) | Aufwand (1 bis 3) | Kategorie | Beleg (Säule) |
|---|---|---|---|---|

Sortiere die Tabelle nach Kategorie. Die drei bis fünf wichtigsten Befunde bilden später den Einstieg in die Strategie-Phase, ohne dass die IST-Analyse die Strategie schon vorwegnimmt.

---

## 3. Aufbau des IST-Berichts

Der Bericht ist beschreibend und belegt, nicht empfehlend. Empfohlene Struktur:

1. **Management Summary** (eine halbe Seite): Wo steht der Auftritt, drei bis fünf Kern-Befunde, jeweils mit Zahl.
2. **Methodik**: Zeitraum, Datenquellen, ER-Formel, Wettbewerberauswahl. Macht den Bericht überprüfbar.
3. **Kanal-Inventar** (aus 01).
4. **Profil-, Visual-, Tonalitäts-Befunde** (aus 01).
5. **Performance-Bestandsaufnahme** (aus 02), mit Zahlen und Belegquelle.
6. **Wettbewerbs-Benchmark** (aus 03), mit Grenzen.
7. **Scorecard** (aus diesem Kapitel).
8. **Priorisierte Baustellen** (Quick Wins zuerst).
9. **Anhang**: Rohdaten-Verweis, Screenshots, Datenstand.

---

## 4. Saubere Übergabe an die Positionierungs-Phase

Die Übergabe ist die Sollbruchstelle zwischen IST und Strategie. Hier endet die Beschreibung, hier beginnt die Empfehlung. Gut übergeben heißt: die nächste Phase bekommt ein belastbares Ausgangsbild plus offene Fragen, aber keine fertigen Antworten.

**Übergabe-Checkliste:**
- [ ] Drei bis fünf priorisierte Baustellen klar benannt, jeweils mit Beleg
- [ ] Datenstand und Zeitraum dokumentiert (damit später Veränderung messbar ist)
- [ ] Benchmark-Position festgehalten (relativ, mit Grenzen)
- [ ] Offene Fragen für die Strategie gesammelt („Warum ist die Zielgruppe so jung?", nicht beantwortet, nur notiert)
- [ ] Rohdaten und Screenshots übergeben und auffindbar
- [ ] Klar markiert, was Befund ist und was bewusst offen bleibt

**Formulierungs-Regel für die Übergabe:** IST-Sätze beginnen mit „Wir beobachten", „Die Daten zeigen", „Im Vergleich zum Set". Sätze, die mit „Wir empfehlen" oder „Wir sollten" beginnen, gehören nicht in den IST-Bericht, sondern in die Phase danach. Diese eine Regel hält die Phasen sauber getrennt.

---

## Output dieser Säule

1. Eine **ausgefüllte Scorecard** über alle Dimensionen und Plattformen.
2. Eine **priorisierte Baustellen-Liste** (Wirkung x Aufwand).
3. Ein **IST-Bericht** nach obiger Struktur.
4. Eine **dokumentierte Übergabe** an die Positionierung.
